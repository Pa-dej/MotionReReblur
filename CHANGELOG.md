# Changelog

Все значимые изменения в этом проекте будут задокументированы в этом файле.

## [1.0.0] - 2024

### Добавлено

#### Совместимость с Iris Shaders
- Добавлен класс `IrisCompat` для определения наличия мода Iris и статуса шейдеров через рефлексию
- Реализована интеграция с Iris API v0 для проверки активных шейдеров и получения имени шейдер-пака
- Экран конфигурации теперь отображает название активного шейдер-пака Iris зеленым цветом
- Motion Blur теперь рендерится параллельно с шейдерами Iris для комбинированных визуальных эффектов

#### Управление глубиной рук
- Добавлен слайдер порога глубины рук в экране конфигурации (диапазон 0.0-1.0)
- Реализованы методы `getHandDepthThreshold()` и `setHandDepthThreshold()` в классе `Module`
- Передача uniform-переменной порога глубины рук в шейдер motion blur для фильтрации по глубине
- Добавлено управление framebuffer'ом рук в `MixinGameRenderer` для изоляции глубины
- Обновлена компоновка экрана конфигурации с подсказками для настройки порога рук

#### Интеграция рендеринга
- Добавлен `MixinGameRenderer` для инъекции применения motion blur перед рендерингом рук
- Извлечена логика motion blur в отдельный метод `applyMotionBlurBeforeHands()` для лучшего контроля
- Применение motion blur теперь происходит после рендеринга мира, но до рендеринга рук и UI
- Расчет FPS перенесен в `applyMotionBlurBeforeHands()` для правильного тайминга

#### Конфигурация проекта
- Добавлены файлы конфигурации IntelliJ IDEA (директория `.idea`)
- Настроены стиль кода, параметры компилятора и интеграция с Gradle
- Добавлены конфигурации запуска для клиента и сервера Minecraft
- Включена библиотека шейдеров Satin в выходные данные сборки
- Обновлен `fabric.mod.json` с метаданными совместимости с Iris

### Изменено

#### Рефакторинг структуры кода
- Переименован `MotionBlurCommand` → `Command` для более чистого API
- Переименован `MotionBlurConfigScreen` → `ConfigScreen` для согласованности
- Переименован `MotionBlurKeyBinding` → `KeyBinding` для краткости
- Переименован `MotionBlurModule` → `Module` для простоты
- Переименован `ShaderMotionBlur` → `Shader` для согласованности
- Обновлены все ссылки в кодовой базе для использования новых имен классов
- Улучшена читаемость кода за счет удаления избыточных префиксов из имен классов

#### Оптимизация рендеринга
- Изменена точка инъекции с `renderHand HEAD` на `afterWorldRender INVOKE`
- Инъекция теперь происходит после вызова `WorldRenderer.render()` вместо перед рендерингом рук
- Добавлен параметр `RenderTickCounter` в callback-метод `afterWorldRender`
- Увеличен приоритет mixin до 1100 в аннотации `MixinGameRenderer`
- Установлен глобальный приоритет mixin 1100 в `motionreblur.mixins.json`
- Рефакторинг регистрации shader callback для применения blur перед рендерингом рук
- Переименован `applyMotionBlurIfNeeded()` → `applyMotionBlurBeforeHands()` для ясности

#### Обновления шейдеров
- Обновлен `motion_blur.fsh` с поддержкой порога глубины рук
- Обновлен `motion_blur.json` с новыми параметрами конфигурации
- Shader.java теперь логирует информацию о шейдер-паке вместо отключения Motion Blur при активном Iris

#### Конфигурация сборки
- Обновлен `build.gradle` с согласованным форматированием зависимостей
- Реорганизованы модификации classpath в конфигурации запуска IDE
- Обновлен `.gitignore` для исключения директории `run/` и `lib/`
- Нормализованы окончания строк в `.gitignore`
- Обновлен `gradle.properties` с конфигурацией проекта
- Обновлена версия Gradle wrapper

### Исправлено
- Предотвращены артефакты размытия рук благодаря правильной точке инъекции рендеринга
- Заменена жестко закодированная ссылка `mc` на `MinecraftClient.getInstance()` для согласованности
- Удалены завершающие пробелы из `Module.java`
- Удален неиспользуемый импорт `ShaderEffectRenderCallback`
